Changelog

A list of highlights for every StopLight release. Not comprehensive.

  • v0.4.58/8/14

    NEW: Support for Link and Unlink HTTP methods.

  • v0.4.48/4/14

    CHANGE: Streamlined app creation. The first environment and endpont are now created automatically for you when you create an app.

    CHANGE: Smarter defaults. When you select an app, the first available environment will be set as active automatically. Auth methods are set by default for every environment (if not already set) when added to an app. SL.variables are automatically created when setting auth methods on apps.

    CHANGE: Underscores and dashes are now allowed in parameter names.

    CHANGE: Numerous bug fixes, including the pesky stuck in the run queue bug.

  • v0.4.37/29/14

    NEW: Support for self-signed SSL certificates.

  • v0.4.27/24/14

    NEW: A tutorial that covers StopLight basics by defining a simple Todo List API.

  • v0.4.17/22/14

    NEW: Manage team members! App admins can add/remove developers and consumers via email. Developers can edit endpoints and tests. Consumers are limited to viewing the app's public documentation, and sending endpoint requests.

  • v0.4.07/16/14

    CHANGE: Updated dashboard layout design. Higher contrast design that makes better use of space.

    CHANGE: Variables are only set if the API responds with a 2xx status code.

    CHANGE: A lot of bug fixes.

  • v0.3.27/3/14

    NEW: In app documentation! icons have been added throughout the StopLight dashboard. Click these to get info on the nearby feature(s).

  • v0.3.16/30/14

    NEW: There's a new clone button on each endpoint in the endpoints list. Use it to erm, clone the endpoint.

    NEW: Add ad-hoc parameters and headers to requests. Just click the little "+" button on the make a request pane.

  • v0.3.06/28/14

    NEW: Demo apps! You'll find these demos on your apps pane, under a new "public" section.

    NEW: App overviews. These are markdown enabled, and give you a place to put high level docs for your app.

    NEW: Save and preview buttons in pane headers.

    NEW: Optional endpoint parameter & header descriptions. To edit a description, click the pencil button next to the parameter on the endpoint edit pane. These are markdown enabled.

    NEW: Syntax highlighting for code in markdown fields.

    CHANGE: Updated the endpoint edit pane design to be easier to understand.

  • v0.2.36/25/14

    NEW: Optional descriptions for endpoint parameters and headers.

    NEW: Underscore.js is now available for use in your test scripts. For example, to check if the response body is empty, you can use _.isEmpty(SL.body).

    CHANGE: Arrays are now supported in variable assignment. For example, you can assign the first value of an array returned by your API like this: SL.variables.firstArrayValueInResponse = SL.body[0].

  • v0.2.26/23/14

    NEW: Authentication Schemes. In the app edit pane, enable the authentication schemes your app supports. Included in this update is authorization via params, headers, and HTTP basic user/password. Use the SL.variables system combined with authentication schemes to easily customize authorization on a per environment basis.

    CHANGE: Better error messages.

  • v0.2.16/22/14

    NEW: Basic themeing options are enabled for all apps. Click on an app to get to the edit page. Here you'll find header color and remote image URL fields.

    CHANGE: Updated, more compact sidebar design.

    CHANGE: The variables viewer now takes up the fill screen.

  • v0.2.06/20/14

    NEW: Endpoint groups make it easy to organize your endpoints. To create one, navigate to the endpoints pane, hover on the caret in the top right, and click "+ Endpoint Group". Drag and drop endpoints into groups. Drag and drop groups to order them.

    CHANGE: Add multiple tests for the same endpoint. This allows you to run a test against endpoint A, then one against endpoint B, and then another one against endpoint A. Navigate to the tests pane and press the "+" in the top right to add a test.

  • v0.1.06/16/14

    Initial release. Ready to rock & roll.
